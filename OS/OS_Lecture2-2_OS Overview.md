# 운영체제의 발전 과정

운영체제는 어떻게 발전해왔는가? 초기에는 운영체제 없이 사용자가 직접 하드웨어를 관리했지만, 배치 시스템을 통해 준비 시간을 줄이고 효율성을 높였으며, 이후 시분할 시스템으로 전환되어 사용자 편의성이 증대되었습니다.

---

## 1. 운영체제 개요 및 발전 과정
운영체제는 컴퓨터 하드웨어를 관리하고 사용자에게 편리한 환경을 제공하며, 시대의 요구에 따라 발전해왔다.

### 1.1. 운영체제 이전 시대와 최초의 운영체제
*   최초의 컴퓨터 시대에는 운영체제가 없었다.
*   사용자는 기계어로 직접 프로그램을 작성해야 했다.
*   운영체제가 담당하는 하드웨어 관리(프로세서, 저장 장치 등)를 사용자가 직접 수행했다.
*   프로그래밍은 매우 어렵고 하드웨어에 대한 깊은 이해가 필요했다.

**프로그램 작성 및 실행 방식**
*   천공 카드라는 구멍을 뚫어 프로그램을 표현하는 방식을 사용했다.
*   사용자는 컴퓨터 앞에 줄을 서서 차례를 기다렸다가 프로그램을 제출하고 결과를 받았다.

**초기 시스템의 비효율성**
*   서로 다른 언어로 작성된 프로그램을 실행하기 위해 컴퓨터를 재설정하는 준비 시간이 필요했다.
*   이러한 준비 시간으로 인해 작업 처리 효율이 떨어지는 단점이 있었다.

### 1.2. 배치 시스템 (일괄 처리 시스템)
**배치 시스템의 등장 배경**
*   개별 작업의 준비 시간을 줄여 컴퓨터의 유휴 시간을 최소화하기 위해 등장했다.

**배치 시스템의 작동 방식**
*   동일한 언어로 작성된 프로그램들을 모아 한 번에 일괄적으로 처리한다.
*   작업 전환 시 발생하는 준비 시간을 한 번으로 줄여 효율성을 높인다.

**배치 시스템의 특징 및 장단점**
*   **시스템 지향적**: 시스템이 처리하기 편하고 효율적으로 자원을 이용할 수 있다.
*   **단점**:
    *   개인 사용자는 작업 제출 후 결과를 받기까지 오랜 시간을 기다려야 한다.
    *   개인 생산성이 저하되고 응답 시간이 길어진다 (예: 6시간).

### 1.3. 시분할 시스템 (타임 쉐어링 시스템)
**시분할 시스템의 등장 배경**
*   배치 시스템의 긴 응답 시간과 개인 사용자의 낮은 생산성 문제를 해결하기 위해 등장했다.

**시분할 시스템의 작동 방식**
*   여러 사용자의 프로그램을 짧은 시간 단위로 나누어 번갈아 실행한다.
*   각 사용자는 마치 자신만 시스템을 사용하는 것처럼 느낄 수 있다.

**시분할 시스템의 특징 및 장단점**
*   **대화형 시스템**: 일정 시간 내에 반응을 보여 사용자와 상호작용이 가능하다.
*   **주요 개념 등장**: 파일 시스템, 가상 메모리 등 사용자 관리를 위한 개념이 발전했다.
*   **장점**:
    *   사용자 측면에서 시스템 사용 편의성과 생산성이 향상된다.
    *   여러 프로세스를 동시에 실행하여 프로세서 활용도를 높인다.
*   **단점**:
    *   통신이 필요하므로 관련 비용이 증가한다.
    *   보안 문제가 발생할 수 있다.
    *   많은 사용자가 접속하면 시스템 부하가 커져 느려질 수 있다.

### 1.4. 개인용 컴퓨터 (PC) 시대
**개인용 컴퓨터의 등장**
*   여러 사용자가 시스템을 공유하는 방식에서 벗어나, 개인이 시스템 전체를 독점하여 사용하는 형태로 발전했다.

**개인용 컴퓨팅 시스템의 특징**
*   **사용자 편리성 중시**: CPU 활용률보다는 사용자가 얼마나 편하게 시스템을 사용하는지가 중요해졌다.
*   **운영체제의 단순화**: 여러 사용자를 관리할 필요가 없어 운영체제가 상대적으로 단순해졌다.
*   **다양한 기능 발전**: 사용자 편의성을 높이는 기능들이 발전했다.
*   **상용화**: 컴퓨터 시스템이 전문화되고 상용화되었다.

**개인용 컴퓨팅 시스템의 단점**
*   개인이 시스템 전체를 사용하므로, 고성능 시스템은 가격이 비싸 구매하기 어렵다.
*   성능이 상대적으로 낮을 수 있다.

### 1.5. 병렬 처리 시스템
**병렬 처리 시스템의 등장 배경**
*   개인용 컴퓨터의 성능 한계를 극복하고 더 높은 성능을 얻기 위해 여러 개의 CPU를 사용하려는 시도에서 시작되었다.

**타이트 커플드 시스템 (Tight-Coupled System)**
*   **정의**: 여러 개의 CPU가 있지만, CPU 외의 다른 자원들은 공유하여 사용하는 시스템이다.
*   **목적**:
    *   기본적으로 성능을 높이는 것이 목적이다.
    *   하나의 CPU가 고장 나더라도 다른 CPU가 대신 작동하여 시스템의 가용성(결함 허용도)을 향상시킨다.
*   **과제**: 프로세서 및 자원 분배와 관리가 복잡해진다.
*   **확장성 제한**: 물리적 공간 및 시스템 제약으로 인해 CPU 개수를 무한정 늘리기 어렵다.

**분산 처리 시스템 (Distributed Processing System)**
*   **정의**: 여러 대의 컴퓨터를 네트워크로 연결하여 하나의 병렬 처리 시스템처럼 사용하는 방식이다.
*   **작동 방식**: 각 컴퓨터(노드)는 자체 운영체제를 가지며, 분산 운영체제를 통해 관리될 수 있다.
*   **장점**:
    *   여러 대의 컴퓨터를 쉽게 연결하여 확장하기 용이하다.
    *   자원을 공유함으로써 높은 성능을 얻을 수 있다.
*   **단점**:
    *   구축 및 유지 보수 관리가 복잡하고 어렵다.
    *   시스템 구성이 복잡해 보인다.
*   **예시**: 클러스터 컴퓨팅, 슈퍼컴퓨터, 클라이언트-서버 시스템 등이 있다.

### 1.6. 실시간 시스템 (Real-Time System)
**실시간 시스템의 정의**
*   작업 처리에 제한 시간을 가지는 시스템이다.
*   요청된 작업에 대해 반드시 정해진 시간 안에 결과를 제공해야 한다.

**실시간 시스템의 종류**
*   **하드 리얼타임 시스템 (Hard Real-Time System)**:
    *   시간 제약을 지키지 못하면 치명적인 문제(재앙)가 발생한다.
    *   예시: 원자력 발전소 제어 시스템, 군사 무기 시스템 등.
*   **소프트 리얼타임 시스템 (Soft Real-Time System)**:
    *   시간 제약을 지키지 못해도 큰 문제나 재난이 발생하지는 않는다.
    *   예시: 유튜브 동영상 재생 (프레임 드롭 발생 시 불편함은 있지만 사회적 재난은 아님).

### 1.7. 운영체제 발전의 요약
*   운영체제의 역할과 작동 방식을 사용자 수, 동시 실행 프로세스 수 등 다양한 관점에서 살펴보았다.
