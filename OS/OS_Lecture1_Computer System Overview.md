# 운영체제 및 컴퓨터 하드웨어 개요

운영체제는 컴퓨터 하드웨어를 효율적으로 관리하여 사용자에게 서비스를 제공하는 소프트웨어입니다. CPU, 메모리, 주변장치 등 다양한 하드웨어 자원을 관리하며, 특히 캐시 메모리는 지역성 원리를 활용하여 메인 메모리와 CPU 간의 속도 차이를 줄여 성능을 향상시키는 핵심적인 역할을 합니다.

---

## 1. 운영체제(OS) 개요

운영체제는 컴퓨터 하드웨어를 효율적으로 관리하여 사용자에게 서비스를 제공하는 소프트웨어다.

### 1.1. 운영체제의 필요성
*   **고성능 컴퓨터를 위한 필수 요소**
    *   게임과 같은 고사양 작업을 위해서는 CPU, GPU, 메모리, 저장 장치 등 다양한 하드웨어가 필요하다.
    *   이러한 하드웨어들을 **효율적으로 활용**하기 위해 운영체제가 필수적이다.

### 1.2. 운영체제의 정의
*   **컴퓨터 하드웨어 관리 및 서비스 제공**
    *   하드웨어 자원을 효율적으로 관리하여 사용자나 응용 프로그램에게 서비스를 제공하는 시스템 소프트웨어다.

---

## 2. 운영체제가 관리하는 컴퓨터 하드웨어

### 2.1. 프로세서 (Processor)
*   **프로세서의 역할 및 종류**
    *   **CPU**: 대표적인 중앙 처리 장치.
    *   **GPU**: 그래픽 및 대량 연산을 위한 장치.
*   **프로세서 내부 구성 요소**
    *   **연산 장치 (ALU)**: 실제 계산 수행.
    *   **제어 장치 (CU)**: 장치의 동작 제어.
    *   **레지스터 (Register)**: 프로세서 내부의 가장 빠른 저장소.
*   **주요 레지스터 종류**
    *   **프로그램 카운터 (PC)**: 다음에 실행할 명령어의 주소를 저장.
    *   **인스트럭션 레지스터 (IR)**: 현재 수행 중인 명령어를 저장.
    *   **누산기 (Accumulator)**: 연산 중간 값을 임시 저장.
*   **운영체제와의 관계**
    *   프로세스에게 프로세서를 할당하고 사용 시간을 관리한다.

### 2.2. 메모리 (Memory)
*   **메모리 계층 구조 (Memory Hierarchy)**
    *   **상위 (빠름, 비쌈, 소용량)**: 레지스터 → 캐시 (L1, L2, L3)
    *   **하위 (느림, 저렴, 대용량)**: 메인 메모리(DRAM) → 보조 기억장치(HDD/SSD)
*   **메인 메모리 (Main Memory)**
    *   CPU와 보조 기억장치 간의 속도 차이를 줄이는 완충 역할.
    *   프로그램 실행을 위해 반드시 데이터가 로드되어야 하는 공간.
*   **캐시 메모리 (Cache Memory)**
    *   **캐시 히트 (Cache Hit)**: 데이터가 캐시에 있어 즉시 사용.
    *   **캐시 미스 (Cache Miss)**: 캐시에 없어 메인 메모리에서 가져옴.
*   **지역성 (Locality) 원리**
    *   **공간적 지역성 (Spatial Locality)**: 참조된 주소 근처를 다시 참조할 확률이 높음 (예: 순차적 배열 접근).
    *   **시간적 지역성 (Temporal Locality)**: 최근 참조된 주소를 곧 다시 참조할 확률이 높음 (예: 반복문).
*   **보조 기억장치 및 가상 메모리**
    *   **가상 메모리 (Virtual Memory)**: 하드디스크의 일부를 메모리처럼 사용하여 메인 메모리 용량 한계를 극복.

### 2.3. 시스템 버스 (System Bus)
*   **버스의 역할**: 프로세서, 메모리, 주변 장치 간의 통로.
*   **버스의 종류**
    *   **데이터 버스**: 실제 데이터 전송.
    *   **주소 버스**: 위치(주소) 정보 전송.
    *   **제어 버스**: 제어 신호(Read/Write 등) 전송.

### 2.4. 주변 장치 (Peripherals)
*   **입출력 장치**: 키보드, 마우스, 모니터 등.
*   **장치 드라이버 (Device Driver)**: 운영체제가 하드웨어를 제어할 수 있도록 제조사가 제공하는 인터페이스.
*   **인터럽트 (Interrupt)**: 주변 장치에서 발생한 이벤트를 운영체제에 알리는 메커니즘.
