# 운영체제 개요

운영체제는 컴퓨터 하드웨어를 효율적으로 관리하고 사용자에게 서비스를 제공하는 핵심 소프트웨어입니다. 운영체제는 사용자 인터페이스 제공, 리소스 관리, 프로세스 관리, 시스템 보호 등 네 가지 주요 역할을 수행하며, 이는 컴퓨터 시스템의 효율성과 안정성을 보장하는 데 필수적입니다.

---

## 1. 운영체제 개요
운영체제는 컴퓨터 하드웨어를 효율적으로 관리하고 사용자에게 서비스를 제공하는 핵심 소프트웨어이다.

### 1.1. 운영체제의 주요 역할
*   **사용자 인터페이스(UI) 제공**
    *   사용자가 컴퓨터 시스템을 편리하게 사용할 수 있도록 돕는다.
    *   과거에는 문자를 기반으로 한 CUI(Character User Interface)를 사용했으나, 현재는 그림 형태의 GUI(Graphical User Interface)가 일반적이다.
    *   특정 목적을 위해 만들어진 기기에는 사용자가 편하게 사용할 수 있도록 특화된 End User Comfortable Interface가 존재할 수 있다. (예: MP3 플레이어)
*   **리소스 관리**
    *   하드웨어 리소스와 소프트웨어 리소스를 관리한다.
    *   주어진 자원을 잘 활용하여 서비스의 효율성을 높이는 역할을 한다.
*   **프로세스 관리**
    *   프로세스는 실행의 주체로, 운영체제는 이러한 프로세스를 관리한다.
    *   스레드와 같은 가벼운 프로세스도 관리한다.
*   **시스템 보호**
    *   사용자의 불법적인 시스템 사용으로부터 시스템을 보호한다.

### 1.2. 컴퓨터 시스템의 구조
*   컴퓨터 시스템은 하드웨어 위에 운영체제(OS)가 존재하고, 그 위에 사용자가 사용하는 응용 프로그램들이 위치하는 계층 구조를 가진다.
*   **커널(Kernel)**: 운영체제의 핵심 부분으로, 하드웨어를 관리하고 사용자에게 서비스를 제공하는 역할을 담당한다.
*   **시스템 콜 인터페이스(System Call Interface)**: 사용자가 커널의 기능을 직접 조작하지 않고 필요한 기능을 요청할 수 있는 통로 역할을 한다. 커널이 제공하는 기능 중 사용자가 사용할 수 있는 기능들을 모아놓은 것이다.

### 1.3. 운영체제 구분 방법
*   **사용자 수에 따른 구분**
    *   **단일 사용자 시스템 (Single User System)**: 한 번에 한 명의 사용자만 시스템을 사용할 수 있다. (예: 윈도우즈 PC, 안드로이드, iOS)
        *   운영체제 관점에서 비교적 간단하게 만들 수 있으며, 자원 관리 및 보호 기능 구현이 용이하다.
    *   **다중 사용자 시스템 (Multi User System)**: 여러 명의 사용자가 동시에 시스템을 사용할 수 있다. (예: 유닉스, 리눅스 서버)
        *   자원에 대한 소유권 관리, 멀티태스킹 기능이 필수적이며 운영체제 기능이 상대적으로 복잡하다.
*   **동시 실행 프로세스 수에 따른 구분**
    *   **단일 작업 시스템 (Single Tasking System)**: 한 번에 하나의 프로그램(프로세스)만 실행할 수 있다. (예: MS-DOS)
        *   하나의 프로그램이 종료되어야 다른 프로그램을 실행할 수 있다.
    *   **멀티태스킹 시스템 (Multi Tasking System)**: 동시에 여러 개의 프로그램(프로세스)을 실행할 수 있다. (예: 윈도우즈, 유닉스)
        *   여러 개의 창을 띄워놓고 작업을 하는 것이 이에 해당한다.
        *   단일 작업 시스템보다 구조가 복잡하다.
